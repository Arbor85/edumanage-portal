# Add Storybook — Implementation Plan

This document is a step-by-step guide for an AI agent. Follow each phase in order.
Each phase lists the exact files to create or modify, props to cover, and mocking strategy.

---

## Governing Rules

- Use **Storybook 8** with `@storybook/vue3-vite` (Vite builder — matches project's Vite setup).
- All stories are written in **TypeScript** using CSF3 format (`satisfies Meta<typeof ...>`).
- Tailwind CSS must be imported in Storybook preview so all styles render correctly.
- Dark mode uses Tailwind `class` strategy — Storybook must include a dark mode toggle that adds/removes `dark` on `<html>`.
- Pinia stores used by Select* components must be seeded with mock data via story decorators.
- API calls must **never fire** in Storybook — mock all stores inline with `setActivePinia(createPinia())` + manual state seeding.
- Never use Auth0 in stories — mock `useAuth0` if any component imports it.
- All story files live next to their component: `src/components/BaseButton/BaseButton.stories.ts` (move component to own folder only if needed — prefer `src/stories/` folder for stories).
- Stories folder: `src/stories/` — all `.stories.ts` files live here.
- Group stories under two top-level categories: **"System / UI"** and **"Business"**.

## Phase 1 — Install & Configure Storybook

### 1.1 Install packages

```bash
npx storybook@latest init --type vue3 --builder vite
npm install -D @storybook/addon-essentials @storybook/addon-interactions @storybook/addon-a11y storybook-dark-mode
```

Expected devDependencies added:
- `@storybook/vue3-vite`
- `@storybook/addon-essentials` (controls, actions, docs, viewport, backgrounds, toolbars)
- `@storybook/addon-interactions`
- `@storybook/addon-a11y`
- `storybook-dark-mode` — toggles `dark` class on `<html>` for Tailwind dark mode

### 1.2 File: `.storybook/main.ts`

```ts
import type { StorybookConfig } from '@storybook/vue3-vite'

const config: StorybookConfig = {
  stories: ['../src/stories/**/*.stories.@(ts|tsx)'],
  addons: [
    '@storybook/addon-essentials',
    '@storybook/addon-interactions',
    '@storybook/addon-a11y',
    'storybook-dark-mode',
  ],
  framework: {
    name: '@storybook/vue3-vite',
    options: {},
  },
  docs: { autodocs: 'tag' },
}
export default config
```

### 1.3 File: `.storybook/preview.ts`

```ts
import type { Preview } from '@storybook/vue3'
import { setup } from '@storybook/vue3'
import { createPinia } from 'pinia'
import '../src/style.css'   // Tailwind base styles

const pinia = createPinia()

setup((app) => {
  app.use(pinia)
})

const preview: Preview = {
  parameters: {
    darkMode: {
      dark: { class: 'dark' },          // adds `dark` to <html>
      stylePreview: true,
    },
    backgrounds: { disable: true },      // using Tailwind bg instead
    layout: 'centered',
    controls: { matchers: { color: /(background|color)$/i, date: /Date$/ } },
  },
}
export default preview
```

### 1.4 Add scripts to `package.json`

```json
"storybook": "storybook dev -p 6006",
"build-storybook": "storybook build"
```

### 1.5 Create `src/stories/` directory

Create placeholder `src/stories/.gitkeep` to establish the folder.

## Phase 6 — Global Mock Strategy

Create `src/stories/mocks/storeData.ts` with factory functions for mock data:

```ts
import type { ClientOut } from '../../types'

export const mockClients: ClientOut[] = [
  { invitationCode: 'c1', name: 'Alice Johnson', email: 'alice@example.com', status: 'active', ... },
  { invitationCode: 'c2', name: 'Bob Smith', email: 'bob@example.com', status: 'inactive', ... },
  { invitationCode: 'c3', name: 'Carol White', email: 'carol@example.com', status: 'active', ... },
]

export const mockExercises = [ ... ]
export const mockRoutines = [ ... ]
export const mockMeetings = [ ... ]
export const mockPlans = [ ... ]
```

Every story that needs API data imports from this file and seeds the store directly.
**No real `axios` calls should happen in any story.**

To prevent axios from firing, add to `.storybook/preview.ts`:
```ts
import axios from 'axios'
axios.defaults.baseURL = 'http://localhost:9999'  // unreachable — will fail loudly if triggered
```
